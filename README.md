# 菁师帮 文档下载捕获器

一个用户脚本（UserScript），在 [菁师帮](https://www.jingshibang.com/) 浏览试卷/文档时，自动捕获其预览接口的真实地址，并通过浏览器直接把文件（PDF / Word）下载到本地。支持**跨页面、跨标签页累进收集**，最后一键全部下载。

> 仅供个人对**自己有权获取**的资料做本地保存（如已登录账号、免费或已购资源）。请遵守菁师帮的服务条款与相关版权规定，自行承担使用责任。脚本本身只在本地拦截 URL 并触发浏览器原生下载，不上传、不收集任何数据。

> 🤖 **本项目是 vibe coding 的产物** —— 脚本与文档均由作者与 [Claude Code](https://claude.com/claude-code)（Anthropic）对话式协作完成。

## 功能

- **自动捕获**：拦截页面对 `/api/public/filepreview` 接口的请求，提取真实文件地址。
- **累进收集**：捕获列表存入 `localStorage`，同源所有标签页共享、刷新不丢；多个详情页（含新开标签页）打开的链接会持续累加到同一个列表。
- **跨标签页实时同步**：任意标签页新增/删除条目，其它标签页的面板会自动刷新。
- **一键全部下载**：把当前收集到的所有文件依次下载。
- **文件名规范化**：
  - 保留正确的扩展名（`.pdf` / `.docx` 等）；
  - 自动区分版本，含答案的（`type=answer`）加 `【答案版】`，避免学生版/答案版同名互相覆盖；
  - 清除文件名中的非法字符（`/ \ : * ? " < > |` 等）。
- **单条操作**：每条链接带“复制链接”“删除”图标按钮。
- **快捷键**：全部下载 / 下载最近一个 / 收起面板。
- **可滚动面板**：条数多时列表区可滚动，不撑爆屏幕（上限 100 条）。

## 安装

1. 给浏览器安装一个用户脚本管理器：
   - [Tampermonkey](https://www.tampermonkey.net/)（Chrome / Edge / Safari / Firefox）
   - 或 [Violentmonkey](https://violentmonkey.github.io/)
2. 打开管理器的“添加新脚本”，把 [`jsb_downloader.js`](jsb_downloader.js) 的全部内容粘贴进去并保存。
3. 访问 `https://www.jingshibang.com/`，脚本会自动生效。

## 使用

1. 打开任意试卷/文档**详情页**（如 `.../paperDetail?id=……`），页面加载预览时，右下角会弹出捕获面板，显示“已捕获 N 个链接”。
2. 想多收集几份，就继续打开其它详情页（**直接开新标签页也可以**），链接会自动累加进同一个列表。
3. 收集完后，在**任意页面**（包括列表主页）点面板里的“全部下载”，或按快捷键，把所有文件一次性下载。
4. 下完后点“清空”重置列表（带二次确认）。

### 面板说明

```
已捕获 3 个链接          [全部下载 ⌥A] [清空]
─────────────────────────────────────────
[下载]  2024北京理工大附中…数学（B卷）【答案版】.pdf   [复制] [删除]
[下载]  2024北京八中高一（下）期末数学.pdf            [复制] [删除]
...
```

- **下载**：下载该条文件。
- **复制**（图标）：复制该条的真实下载链接，复制成功会短暂变成 ✓。
- **删除**（图标）：把该条从列表移除（不影响其它条目）。

### 快捷键

| Windows / Linux | macOS | 作用 |
|-----------------|-------|------|
| `Alt + A` | `⌥ Option + A` | 全部下载 |
| `Alt + D` | `⌥ Option + D` | 下载最近捕获的一个 |
| `Alt + H` | `⌥ Option + H` | 显示 / 隐藏面板 |

## 工作原理

脚本在页面最早期（`@run-at document-start`）重写 `window.fetch` 与 `XMLHttpRequest.prototype.open`，只“旁听”请求 URL、不修改请求；命中 `filepreview` 接口时记录该地址，解析出规范文件名，存入 `localStorage` 并渲染面板。下载通过创建带 `download` 属性的 `<a>` 元素触发，从而让浏览器**保存文件而非在线预览**。

---

## 注意事项

- **首次“全部下载”浏览器会弹一次“是否允许下载多个文件”**，点允许即可——这是浏览器安全机制。多个文件按约 600ms 间隔依次触发。
- **localStorage 按域名隔离**：若站点在 `www.jingshibang.com` 与 `jingshibang.com`（无 www）之间混用，两边的累积列表不互通。
- **个别文件可能没有扩展名**：当接口的 `filepath` 本身不带后缀时，脚本无法可靠判断是 PDF 还是 Word，因而不会强加后缀，存下来的文件可能需要手动补 `.pdf` / `.docx`。
- 列表上限 100 条，超出后最旧的会被丢弃。
